# 中医处方管理数据库系统 - 需求文档

## 1. 项目概述

### 1.1 项目背景
本项目为湖北中医药大学留学生计算机应用期末考试试题，要求学生使用Access 2016或更高版本完成一个完整的中医处方管理数据库系统。该系统用于管理中医科室、医师、患者及处方信息，实现处方数据的增删改查、统计分析与报表打印功能。

### 1.2 项目目标
- 建立一个规范化的中医处方管理数据库
- 实现医师信息管理、处方数据录入与查询
- 设计友好的用户界面（表单）用于数据浏览与操作
- 生成标准化的处方报表供打印输出
- 满足考试评分标准中的所有技术要求

### 1.3 用户角色
- **系统管理员**：负责数据库的创建、表结构设计与关系建立
- **医师用户**：浏览个人基本信息，查看所开处方记录
- **数据操作员**：录入和维护处方数据，执行各类查询

## 2. 功能需求

### 2.1 数据库基础操作模块（20分）
| 功能点 | 详细要求 | 分值 |
|--------|----------|------|
| 1. 数据导入 | 将`TCM_Prescription_Information.txt`文件导入数据库，新建`TCM_Prescription_Table`表，设置字段数据类型（`Physician_ID`为文本，`Patient_Gender`为是/否），确保无重复记录 | 5分 |
| 2. 表结构修改 | 按指定要求修改`TCM_Prescription_Table`表结构，包括字段名、数据类型、字段大小和备注 | 3分 |
| 3. 主键与索引 | 设置`Prescription_ID`为主键，为`Formula_Name`字段创建普通索引 | 4分 |
| 4. 关系与参照完整性 | 在`TCM_Prescription_Table`与`Doctor_Table`之间建立主-从关系（通过`Physician_ID`字段），设置级联更新和级联删除 | 4分 |
| 5. 数据插入 | 向`Doctor_Table`插入新记录，包含科室ID、医师ID（学号后六位）、医师性别、年龄、职称、学历等个人信息，照片字段留空 | 4分 |

### 2.2 查询设计模块（20分）
| 查询名称 | 功能描述 | 技术要求 | 分值 |
|----------|----------|----------|------|
| 1. 统计医师处方数量与金额 | 统计姓名恰好为三个汉字的医师的处方数量和金额小计，显示医师姓名、处方数量、处方小计 | 保存为`1_Count_Physician_Prescriptions_and_Amounts` | 5分 |
| 2. 按年龄范围查询患者 | 提示用户输入最小年龄和最大年龄（示例输入：25和40），将25-40岁患者的姓名、性别、年龄、临床诊断保存到新表`Patients Aged 25 to 40` | 保存为`2_Query_Patients_by_Age_Range` | 5分 |
| 3. 调整处方金额 | 将`TCM_Prescription_Table`中金额低于150元的处方金额提高5% | 保存为`3_Increase_Prescription_Amounts` | 5分 |
| 4. 统计科室医师患者性别分布 | 统计各科室各医师治疗的不同性别患者数量，以交叉表形式显示（科室名、医师名为行标题，性别为列标题） | 保存为`4_Count_Patients_by_Department_and_Physician` | 5分 |

### 2.3 表单设计模块（30分）
#### 2.3.1 医师基本信息管理表单
- **表单标题**：医师基本信息管理
- **打开方式**：弹出式对话框，边框样式为"对话框"
- **界面要求**：无导航按钮、记录选择器、分割线、滚动条；控制框无最大化/最小化按钮
- **标题标签**：标签文本为"医师基本信息管理"，字体为隶书、22号、加粗、红色、居中
- **显示字段**：医师ID、科室名称、医师姓名、医师性别、医师年龄、医师照片、医师学历、医师职称
- **装饰要求**：表单细节区域顶部和底部设置红色实线边框，宽度3磅
- **按钮功能**：添加五个命令按钮：首记录、上一条、下一条、末记录、关闭表单
- **布局要求**：注意控件美观布局

#### 2.3.2 浏览与打印处方信息表单
- **表单标题**：浏览与打印处方信息
- **打开方式**：弹出式对话框，边框样式为"对话框"，无导航按钮、记录选择器、分割线、滚动条；控制框无最大化/最小化按钮
- **标题标签**：标签文本为"浏览与打印处方信息"，字体为STXinwei、24号、加粗、蓝色、居中
- **设计师标签**：在"刷新"按钮后添加"设计师：[考生姓名]"，字体为楷体、12号、加粗、红色
- **功能流程**：
  1. 通过"选择医师"下拉框选择医师
  2. 点击"刷新"按钮，在子窗体中显示当前医师开具的处方基本信息
  3. 点击"预览医师处方列表"按钮，以打印预览模式显示"打印医师处方列表"报表
- **子窗体显示字段**：患者姓名、临床诊断、方剂名称、处方日期、处方金额（仅五字段）

### 2.4 综合设计模块（30分）
#### 2.4.1 打印医师处方列表报表
- **报表标题**："[医师姓名]处方列表"，包含所选医师的姓名和职称，字体为宋体、24号、加粗、红色
- **统计信息**：报表页脚统计处方数量和总金额，显示"总计：xx处方"和"累计处方金额：xx元"，字体为宋体、14号、加粗、红色
- **处方信息框**：每条处方信息置于矩形框内，边框宽度3磅、蓝色
- **字段格式**：
  - `Prescription_ID`：宋体、18号、黑色
  - 其他字段：宋体、12号、黑色
  - `Patient_Age`显示为"X岁"
  - `Prescription_Amount`显示为"¥X元"
- **区域分隔**：处方ID、处方正文、处方附注部分用虚线分隔，样式为"点划线"，宽度1磅，黑色
- **布局要求**：注意报表整体美观性

## 3. 非功能需求

### 3.1 技术环境
- **数据库平台**：Microsoft Access 2016或更高版本
- **文件格式**：所有考试材料保存在"考试材料"文件夹中，需重命名为包含学号和姓名的格式
- **提交要求**：将整个文件夹压缩为.rar或.zip文件，在2026年3月27日前提交至指定邮箱

### 3.2 性能要求
- 查询响应时间应在合理范围内（无具体指标）
- 表单加载和切换应流畅
- 报表生成和打印预览应正常运作

### 3.3 界面要求
- 所有表单采用对话框样式，界面简洁
- 字体、颜色、大小严格按考试要求设置
- 控件布局美观、对齐整齐

### 3.4 数据完整性
- 主键约束确保处方ID唯一
- 参照完整性确保处方表中的医师ID必须在医师表中存在
- 级联更新和级联删除保护数据一致性

### 3.5 安全性
- 考生需使用个人学号和姓名相关信息
- 如使用他人信息，考试成绩按0分处理

## 4. 验收标准
- 每个功能点按考试分值评分
- 所有查询、表单、报表需按指定名称保存
- 数据库关系、索引设置正确
- 界面样式严格符合题目要求
- 最终作品打包提交，包含完整数据库文件
