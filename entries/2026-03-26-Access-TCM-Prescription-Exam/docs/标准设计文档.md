## 3. 数据库设计

### 3.1 数据库概念模型
系统采用关系型数据库模型，包含三个核心实体：
1. **科室实体**: 医院科室信息
2. **医生实体**: 医生基本信息
3. **处方实体**: 中医处方记录

实体间关系：
- 一个科室包含多名医生 (1:N)
- 一名医生可开具多张处方 (1:N)

## 4. 查询功能设计

### 4.1 查询设计原则
- **功能性**: 满足所有查询需求
- **效率性**: 合理使用索引优化查询性能
- **友好性**: 参数查询提供明确提示信息
- **规范性**: 查询命名符合要求

### 4.2 查询功能列表

#### 4.2.1 查询1: 医生处方统计查询
- **查询名称**: `1_Count_Physician_Prescriptions_and_Amounts`
- **功能描述**: 统计姓名恰好为三个汉字的医生开具的处方数量和总金额
- **SQL语句**:
  ```sql
  SELECT 
      d.Physician_Name AS 医生姓名,
      COUNT(p.Prescription_ID) AS 处方数量,
      SUM(p.Prescription_Amount) AS 处方小计
  FROM 
      Doctor_Table d
      INNER JOIN TCM_Prescription_Table p 
      ON d.Physician_ID = p.Physician_ID
  WHERE 
      LEN(d.Physician_Name) = 3
  GROUP BY 
      d.Physician_Name;
  ```
- **输出字段**: 医生姓名, 处方数量, 处方小计

#### 4.2.2 查询2: 患者年龄范围查询
- **查询名称**: `2_Query_Patients_by_Age_Range`
- **功能描述**: 根据用户输入的年龄范围查询患者信息并生成新表
- **参数设置**:
  - 最小年龄参数: `[请输入最小年龄。]`
  - 最大年龄参数: `[请输入最大年龄。]`
- **目标表**: `Patients_Aged_25_to_40` (当输入25和40时)
- **输出字段**: 患者姓名, 性别, 年龄, 临床诊断

#### 4.2.3 查询3: 处方金额更新查询
- **查询名称**: `3_Increase_Prescription_Amounts`
- **功能描述**: 将金额低于150元的处方增加5%
- **SQL语句**:
  ```sql
  UPDATE TCM_Prescription_Table
  SET Prescription_Amount = Prescription_Amount * 1.05
  WHERE Prescription_Amount < 150;
  ```
- **影响范围**: 所有金额低于150元的处方记录

#### 4.2.4 查询4: 患者性别交叉统计
- **查询名称**: `4_Count_Patients_by_Department_and_Physician`
- **功能描述**: 按科室、医生、性别统计患者数量(交叉表格式)
- **行标题**: 科室名称, 医生姓名
- **列标题**: 患者性别
- **值**: 患者计数
- **统计方式**: 交叉表查询

### 4.3 查询实现技术
1. **参数查询实现**: 在查询条件中使用`[参数名]`格式
2. **生成表查询**: 使用`SELECT ... INTO`语句
3. **更新查询**: 使用`UPDATE ... SET ... WHERE`语句
4. **交叉表查询**: 使用Access交叉表查询向导
