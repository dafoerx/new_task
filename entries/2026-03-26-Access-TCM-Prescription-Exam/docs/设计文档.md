# 中医处方管理数据库系统 - 设计文档

## 1. 数据库设计

### 1.1 概念模型
系统包含三个核心实体：
- **科室**：医院的中医科室，如内科、妇科等
- **医师**：各科室的中医医师，具有职称、学历等属性
- **处方**：医师为患者开具的中医处方，包含方剂、剂量、金额等信息

实体关系：一个科室有多个医师，一个医师开具多个处方。

|--------|----------|----------|------|------|

| Department_ID | 文本 | 10 | 是 | 科室编号，外键 |
| Physician_Name | 文本 | 20 | 是 | 医师姓名 |
| Physician_Gender | 文本 | 2 | 是 | 医师性别（男/女） |
| Physician_Age | 数字 | 整型 | 否 | 医师年龄 |
| Physician_Photo | OLE对象 | - | 否 | 医师照片 |
| Physician_Education | 文本 | 20 | 否 | 医师学历 |
| Physician_Title | 文本 | 20 | 否 | 医师职称 |

#### 1.2.3 处方表 (TCM_Prescription_Table)
| 字段名 | 数据类型 | 字段大小 | 必填 | 说明 |
|--------|----------|----------|------|------|
| Prescription_ID | 文本 | 10 | 是 | 处方编号，主键 |

| Patient_Gender | 是/否 | - | 是 | 患者性别（0:男，-1:女） |
| Patient_Age | 数字 | 整型 | 否 | 患者年龄 |
| Clinical_Diagnosis | 文本 | 30 | 是 | 临床诊断 |

| Formula_Composition | 备注 | - | 否 | 方剂组成 |
| Physician_ID | 文本 | 10 | 是 | 医师编号，外键 |
| Prescription_Date | 日期/时间 | - | 是 | 处方日期 |
| Prescription_Amount | 货币 | - | 是 | 处方金额 |

### 1.3 关系设计
1. **Department_Table** ↔ **Doctor_Table**  
   - 关系类型：一对多
   - 关联字段：Department_Table.Department_ID → Doctor_Table.Department_ID
   - 参照完整性：强制

2. **Doctor_Table** ↔ **TCM_Prescription_Table**  
   - 关系类型：一对多
   - 关联字段：Doctor_Table.Physician_ID → TCM_Prescription_Table.Physician_ID
   - 参照完整性：级联更新相关字段、级联删除相关字段

### 1.4 索引设计
| 表名 | 索引字段 | 索引类型 | 说明 |
|------|----------|----------|------|
| TCM_Prescription_Table | Prescription_ID | 主键 | 唯一索引 |
| TCM_Prescription_Table | Formula_Name | 普通索引 | 提高查询性能 |

## 2. 查询设计

### 2.1 查询1：统计医师处方数量与金额
```sql
-- Access SQL 示例
SELECT 
    D.Physician_Name AS 医师姓名,
    COUNT(P.Prescription_ID) AS 处方数量,
    SUM(P.Prescription_Amount) AS 处方小计
FROM 
    Doctor_Table AS D
    INNER JOIN TCM_Prescription_Table AS P ON D.Physician_ID = P.Physician_ID
WHERE 
    LEN(D.Physician_Name) = 3  -- 假设姓名为三个汉字
GROUP BY 
    D.Physician_Name
ORDER BY 
    处方数量 DESC;
```

**保存名称**：`1_Count_Physician_Prescriptions_and_Amounts`

### 2.2 查询2：按年龄范围查询患者
```sql
-- 参数查询设计
PARAMETERS [请输入最小年龄:] Long, [请输入最大年龄:] Long;
SELECT 
    Patient_Name AS 患者姓名,
    Patient_Gender AS 性别,
    Patient_Age AS 年龄,
    Clinical_Diagnosis AS 临床诊断
INTO 
    [Patients Aged 25 to 40]
FROM 
    TCM_Prescription_Table
WHERE 
    Patient_Age BETWEEN [请输入最小年龄:] AND [请输入最大年龄:];
```

**保存名称**：`2_Query_Patients_by_Age_Range`

### 2.3 查询3：调整处方金额
```sql
-- 更新查询设计
UPDATE 
    TCM_Prescription_Table
SET 
    Prescription_Amount = Prescription_Amount * 1.05
WHERE 
    Prescription_Amount < 150;
```

**保存名称**：`3_Increase_Prescription_Amounts`

### 2.4 查询4：统计科室医师患者性别分布
```sql
-- 交叉表查询设计
TRANSFORM 
    COUNT(T.Prescription_ID) AS 患者数量
SELECT 
    D.Department_Name AS 科室名称,
    D.Physician_Name AS 医师姓名
FROM 
    (Department_Table AS Dept
    INNER JOIN Doctor_Table AS D ON Dept.Department_ID = D.Department_ID)
    INNER JOIN TCM_Prescription_Table AS T ON D.Physician_ID = T.Physician_ID
GROUP BY 
    D.Department_Name, D.Physician_Name
PIVOT 
    T.Patient_Gender;
```

**保存名称**：`4_Count_Patients_by_Department_and_Physician`

## 5. 系统架构建议

### 5.1 技术栈
- **数据库管理系统**：Microsoft Access 2016+
- **开发工具**：Access内置设计器（表设计、查询设计、表单设计、报表设计）
- **编程语言**：VBA（如需扩展功能）
- **文件格式**：.accdb（Access数据库文件）

### 5.2 部署方案
1. **开发环境**：在个人计算机上使用Access进行开发
2. **测试环境**：在考试要求的"考试材料"文件夹中测试所有功能
3. **交付物**：将整个文件夹重命名为"学号_姓名_考试材料"，压缩为.rar或.zip文件
4. **提交方式**：电子邮件提交至指定邮箱

### 5.3 扩展建议
- **数据备份**：定期备份.accdb文件
- **性能优化**：为常用查询字段创建索引
- **安全增强**：设置数据库密码保护
- **多语言支持**：可考虑添加英文界面（针对留学生需求）
