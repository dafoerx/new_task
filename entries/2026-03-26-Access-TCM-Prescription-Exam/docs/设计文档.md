## 1. 系统概述

### 1.1 项目背景
本项目是湖北中医药大学留学生计算机应用课程的期末考试项目，旨在测试学生使用Access数据库管理系统进行完整应用开发的能力。系统需实现中医处方管理的全流程功能。

### 1.2 系统目标
- 实现中医处方数据的规范化管理
- 提供医生信息管理功能
- 支持多样化的数据查询和统计
- 提供友好的用户界面和报表输出
- 满足考试评分标准中的所有技术要求

### 1.3 系统架构
```
中医处方管理数据库系统
├── 数据库层（Access Tables）
│   ├── 科室表 (Department_Table)
│   ├── 医生表 (Doctor_Table) 
│   └── 中医处方表 (TCM_Prescription_Table)
├── 业务逻辑层（Queries）
│   ├── 统计查询
│   ├── 参数查询
│   ├── 更新查询
│   └── 交叉表查询
├── 表示层（Forms）
│   ├── 医生基本信息管理表单
│   └── 处方浏览打印表单
└── 输出层（Reports）
    └── 医生处方列表报表
```

## 3. 查询设计

### 3.1 查询列表

#### 3.1.1 查询1：医生处方统计
- **查询名称**: `1_Count_Physician_Prescriptions_and_Amounts`
- **功能**: 统计姓名恰好为三个汉字的医生开具的处方数量和总金额
- **输出字段**: 医生姓名、处方数量、处方小计
- **SQL逻辑**:
  ```sql
  SELECT 
      d.Physician_Name,
      COUNT(p.Prescription_ID) AS 处方数量,
      SUM(p.Prescription_Amount) AS 处方小计
  FROM 
      Doctor_Table d
      INNER JOIN TCM_Prescription_Table p ON d.Physician_ID = p.Physician_ID
  WHERE 
      LEN(d.Physician_Name) = 3
      AND d.Physician_Name NOT LIKE '%[0-9]%'  -- 确保是汉字
  GROUP BY 
      d.Physician_Name;
  ```

#### 3.1.2 查询2：年龄范围患者查询
- **查询名称**: `2_Query_Patients_by_Age_Range`
- **功能**: 根据输入的年龄范围查询患者信息并生成新表
- **参数提示**: "请输入最小年龄。" 和 "请输入最大年龄。"
- **输出表**: `Patients_Aged_25_to_40`（当输入25和40时）
- **输出字段**: 患者姓名、性别、年龄、临床诊断

#### 3.1.3 查询3：处方金额更新
- **查询名称**: `3_Increase_Prescription_Amounts`
- **功能**: 将金额低于150元的处方增加5%
- **更新逻辑**:
  ```sql
  UPDATE TCM_Prescription_Table
  SET Prescription_Amount = Prescription_Amount * 1.05
  WHERE Prescription_Amount < 150;
  ```

#### 3.1.4 查询4：科室医生患者交叉统计
- **查询名称**: `4_Count_Patients_by_Department_and_Physician`
- **功能**: 按科室、医生、性别统计患者数量（交叉表）
- **行标题**: 科室名称、医生姓名
- **列标题**: 患者性别
- **值**: 患者计数

### 3.2 查询实现要点
1. **参数查询**: 使用Access的参数提示功能
2. **生成表查询**: 使用`SELECT ... INTO`语句
3. **更新查询**: 注意更新条件的准确性
4. **交叉表查询**: 使用Access的交叉表查询向导

## 6. 实现步骤

### 6.1 第一阶段：数据库基础操作（预计时间：2-3小时）
1. **准备工作**
   - 打开Access 2016+
   - 创建新数据库文件，命名为"TCM Prescription Management Database.accdb"
   - 确认已有`Department_Table`和`Doctor_Table`

2. **数据导入**
   - 使用"外部数据" → "文本文件"导入`TCM_Prescription_Information.txt`
   - 指定新表名为`TCM_Prescription_Table`
   - 在导入向导中设置`Physician_ID`为文本类型，`Patient_Gender`为是/否类型
   - 勾选"不重复记录"选项

3. **表结构修改**
   - 按照2.1.3节的要求修改`TCM_Prescription_Table`的字段属性

   - 在`Formula_Name`字段上创建常规索引"Name_Prescription"

4. **关系建立**
   - 打开"数据库工具" → "关系"

5. **数据插入**
   - 向`Doctor_Table`插入新记录，包含学生个人信息

## 7. 技术难点与解决方案

### 7.1 技术难点
1. **交叉表查询**：Access交叉表查询的语法和设置
2. **参数查询**：如何实现运行时输入参数
3. **表单格式**：精确控制字体、颜色、边框等属性
4. **报表格式**：复杂格式要求，特别是虚线分隔和动态标题

### 7.2 解决方案
1. **交叉表查询**：使用查询设计视图的"交叉表查询"模式
2. **参数查询**：在查询设计视图中右键选择"参数"设置
3. **表单格式**：通过属性表详细设置每个控件的格式属性
4. **报表格式**：使用报表设计视图的节和控件属性设置

### 7.3 常见错误预防
1. **字段类型错误**：导入时仔细设置字段类型
2. **关系错误**：确保外键字段类型与主键完全一致
3. **字体缺失**：使用Windows标准字体或确认字体已安装
4. **压缩错误**：确认所有文件都在同一文件夹中再压缩
