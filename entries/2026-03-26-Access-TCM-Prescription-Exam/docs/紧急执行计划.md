## 分阶段详细执行指南

### 阶段1：数据库基础操作（目标：2-3小时完成）
**关键操作步骤**：
1. **打开Access，创建新数据库**（10分钟）
   - 文件 → 新建 → 空白数据库
   - 保存为"TCM Prescription Management Database.accdb"

2. **导入TXT数据**（30分钟）
   - 外部数据 → 文本文件
   - 选择"TCM_Prescription_Information.txt"
   - 重要：在导入向导中：
     - `Physician_ID`字段 → 设置为"文本"
     - `Patient_Gender`字段 → 设置为"是/否"
     - 勾选"不重复记录"
   - 表名设为"TCM_Prescription_Table"

3. **修改表结构**（30分钟）
   - 在设计视图中打开`TCM_Prescription_Table`
   - 按设计文档2.1.3节的要求设置字段
   - 设置`Prescription_ID`为主键
   - 在`Formula_Name`字段创建索引，命名为"Name_Prescription"

4. **建立关系**（20分钟）
   - 数据库工具 → 关系
   - 添加`Doctor_Table`和`TCM_Prescription_Table`
   - 拖拽`Physician_ID`字段建立关系
   - 勾选"实施参照完整性"、"级联更新相关字段"、"级联删除相关记录"

5. **插入学生记录**（10分钟）
   - 打开`Doctor_Table`的数据表视图
   - 插入新记录：`Department_ID`为"K1"，`Physician_ID`为学号后6位
   - 填写个人其他信息，`Photo`字段留空

**检查点**：
- [ ] 数据导入成功，无错误
- [ ] `Patient_Gender`字段显示为复选框
- [ ] `Prescription_ID`有钥匙图标（主键）
- [ ] 关系图中看到连接线
- [ ] 学生记录已插入

### 阶段2：查询设计（目标：3-4小时完成）
**查询执行顺序**：
1. **查询2：年龄参数查询**（最容易，先建立信心）
2. **查询3：金额更新查询**（简单，但注意备份）
3. **查询1：医生处方统计**（需要连接表）
4. **查询4：交叉表查询**（最复杂，最后做）

**关键技巧**：
- **参数查询**：在设计视图中，在条件行输入`[请输入最小年龄。]`
- **更新查询**：先创建选择查询验证条件，再转为更新查询
- **交叉表查询**：使用查询向导的"交叉表查询"选项
- **保存查询**：严格按要求的名称保存

**检查点**：
- [ ] 查询2运行时提示输入年龄
- [ ] 查询3执行后，金额低于150的处方增加5%
- [ ] 查询1只显示三个汉字姓名的医生
- [ ] 查询4生成行列交叉的统计表
